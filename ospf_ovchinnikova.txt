E2 - External Type 2
Значения метрик не складываются, а используется только метрика объявляемая из внешней сети. Рассчитывается стоимость перехода из одной АС в другую.

2. Объяснить смысл данных выводимых командой show ip ospf database external. 
Выводится адреса, маски, ID роутера и другая информация о внешних сетях, которые используют другие протоколы маршрутизации.

3. Кто и каким образом генерирует External LSA? Какую информацию содержат External LSA? 
Маршрутизатор R2AR0 (пограничный) рассылает OSPF пакеты во все возможные области. Они содержат описание внешних маршрутов, которые передаются в OSPF.

1.	Что такое тупиковая область? Для чего нужна? 
Не принимает информацию о внешних маршрутах для АС, но принимает маршруты из других зон.
Для передачи информации за границу автономной системы используют маршрут по умолчанию.
Позволяет снизить нагрузку на маршрутизаторы внутри области, из-за расчета меньшего количества маршрутов.

2. Как отразилось на таблице маршрутизации и содержимом топологической базы маршрутизаторов ABR и R1AR1 то, что область стала тупиковой? 
У роутера R1AR1 пропала вся информация о внешних маршрутах (External LSA), и появился маршрут по умолчанию (Summary LSA)

На маршрутизаторе R1AR1 видны сети, помеченные кодом «N» (NSSA External)

ABR-роутер выполняет конвертацию NSSA External в External и передает их в другие области.
